# 決算分析Webアプリ 設計図

## 1. アプリの概要
企業の決算情報や株価、金利のデータを元に、ネットキャッシュ比率やキャッシュニュートラルPERなどの高度な財務指標を自動計算・表示するWebアプリケーション。
Viteなどの複雑な技術は使わず、HTML / CSS / JavaScript のみを使用し、初心者でも構築・改修しやすいシンプルな設計とする。

## 2. 実装のステップ（スモールスタートの提案）

**提案事項：** いちからWebアプリを制作する場合、他サイトの情報の直接取得や、ブラウザの通信制限（CORSエラー）、そしてPDFの直接解析は非常に難易度が高く、挫折の原因になりやすいです。そのため、以下のようにフェーズを分けて確実に実装していくアプローチをおすすめします。

### フェーズ1：完全手入力の計算ツール（最優先で実装）
外部サイトへの通信やPDF解析の機能はいったん省き、計算ロジックと画面の完成に集中します。
- ユーザーがフォームに「売上高」「時価総額」「株価」「金利」などを手入力する。
- 「計算する」ボタンを押すと、JavaScriptで指定された計算式に則り、各指標が瞬時に計算され画面に表示される。

### フェーズ2：テキスト貼り付けによる入力の自動化（LLMの活用）
- 決算短信のPDFを直接読み込むのではなく、PDFのテキストをコピーしてアプリ内の「テキストエリア」に貼り付ける仕様にします。
- コピーした文章をLLM（生成AI）のAPIに渡し、「売上高や自己資本などの数値をJSONで抽出して」と指示することで、フォームに自動で数値が埋まる仕組みを作ります。

### フェーズ3：株価・金利の自動取得（バックエンドの導入）
- Yahoo!ファイナンスや楽天証券からブラウザのJavaScriptで直接スクレイピングしようとすると、ブラウザのセキュリティ機能（CORS）でブロックされます。
- これを解決するために、Google Apps Script (GAS) などの無料で簡単に作れる中継APIを作成し、そこ経由で最新の株価や金利を取得するように拡張します。（あるいはここも最初は手入力で妥協します）。

## 3. 計算ロジック・取得項目

### 【入力・取得が必要なデータ】
- 売上高（億単位）
- 時価総額（億単位）
- 流動資産
- 流動負債（※流動比率の計算に使用）
- 在庫
- 投資有価証券
- 負債（総負債）
- 自己資本
- 資産合計（※自己資本比率の計算に使用）
- 営業利益
- 当期純利益（※ROEの計算に使用）
- 現在の株価
- 予想EPS / 実績EPS（※PERの計算に使用。またはEPSの代わりにすでに計算済みの実績PER/予測PERを入力してもらう）
- BPS（※PBRの計算に使用。または 時価総額と自己資本から計算）
- 想定金利（日本国債10年物の利回り）

### 【計算式】
- **PSR** = 時価総額 / 売上高
- **流動比率** = 流動資産 / 流動負債
- **在庫比率** = 在庫 / 流動資産
- **自己資本比率** = 自己資本 / 資産合計
- **営業利益率** = 営業利益 / 売上高
- **ROE** = 当期純利益 / 自己資本
- **PBR** = 株価 / BPS （または 時価総額 / 自己資本）
- **PER（実績）** = 株価 / 実績EPS
- **PER（予測）** = 株価 / 予想EPS
- **ネットキャッシュ比率** = （流動資産 - 在庫 + 投資有価証券 × 0.7 - 負債） / 時価総額
  - ※在庫がない場合は0として計算
- **キャッシュニュートラルPER（実績）** = （1 - ネットキャッシュ比率） × PER（実績）
- **キャッシュニュートラルPER（予測）** = （1 - ネットキャッシュ比率） × PER（予測）
- **想定利益成長率** = ((キャッシュニュートラルPER（予測） - 1) × 想定金利 - 1) / キャッシュニュートラルPER（予測）

## 4. セキュリティ設計

初心者がネットに公開してもまったく問題ない、安全な作りにするために以下の対策を設計に組み込みます。

### 1. APIキーのハードコード（直接記述）禁止 【最重要】
- **危険な構造**: JavaScriptコードの中に直接 `const API_KEY = "sk-..."` と書いてしまうこと。GitHubなどでソースコードを公開すると、瞬時に世界中のBotに検知され他人に不正利用されて高額請求が来るリスクがあります。
- **セキュアな設計（ブラウザ入力方式）**: アプリの画面上に「APIキー設定（歯車マークなど）」のメニューを作ります。ユーザー自身がブラウザの画面から自分のAPIキーを入力し、ブラウザのローカル（`localStorage`）にのみ保存させます。プログラム内にはキーを一切書き込みません。

### 2. クロスサイトスクリプティング（XSS）対策
- 決算短信のテキストやAIからの返答を画面に表示する際、悪意のあるコードが含まれていた場合に実行させないようにします。
- JavaScriptでHTMLを書き換えるときは、`innerHTML` を**絶対に使わず**に、必ず `textContent` を使用して安全なただの文字列として描画します。

### 3. HTTPSでの公開
- GitHub Pages や Vercel など、HTTPS通信（暗号化）が標準で提供されている無料のホスティングサービスを利用してサイトを公開し、通信傍受を防ぎます。
